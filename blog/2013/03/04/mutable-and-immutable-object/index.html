python mutable and immutable object

Mar 4th, 2013

there are two types in python: mutable and immutable. tuple, number, string are all immutable. dictionary and list are mutable.

passing parameters

coders can not decide whether to pass by value or pass by reference. for mutable object, passing by value. for immutable object, pass by reference.

callid objName to watch memory address

Default parameter values are evaluated when the function definition is executed. This means that the expression is evaluated once, when the function is defined, and that the same “pre-computed” value is used for each call. This is especially important to understand when a default parameter is mutable, such as a list or a dictionary: if the function modifies the object(e.g. by appending an item to a list), the default value is in effect modified. This is generally not what was intended. A way around this is to use None as the default, and explicitly test for it in the body of the function.
